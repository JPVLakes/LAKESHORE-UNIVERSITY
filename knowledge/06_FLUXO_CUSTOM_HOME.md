# Fluxo End-to-End: Custom Home - Lakeshore Development

## Visao Geral

Uma **Custom Home** e uma casa totalmente personalizada, construida a partir das especificacoes de um cliente especifico. Diferente da Spec Home, a Custom Home comeca com o relacionamento com o cliente — antes de qualquer terreno, planta ou orcamento. O cliente geralmente ja tem ou tem opcao de compra de um terreno e deseja uma casa projetada sob medida.

Este fluxo cobre o ciclo completo: da captacao do cliente ate a entrega da casa e o periodo de garantia pos-venda. Cada etapa identifica o departamento responsavel, as acoes principais, o handoff para a proxima etapa e as ferramentas utilizadas.

**Diferenca-chave em relacao a Spec Home:**
- Começa com Client Relations / Escritorio de Projetos (nao com Land)
- Projeto e orcamento sao unicos e podem mudar multiplas vezes
- Sem templates replicaveis: cada casa e um projeto do zero
- PM (Project Manager) assume gestao apos assinatura do contrato
- Compras sao gerenciadas pelo proprio PM de custom (nao ha setor de compras dedicado)

## Etapa 2: Escritorio de Projetos - Desenvolvimento do Projeto

| Campo | Detalhe |
|-------|---------|
| **Departamento** | Escritorio de Projetos (Camila Martins / equipe de arquitetura) |
| **Gatilho de inicio** | Pre-agreement assinado e dados do cliente/terreno recebidos |
| **Ferramentas** | AutoCAD, SketchUp, Enscape/Lumion, Google Drive, ClickUp |

**Acoes principais:**
1. Reuniao com cliente para detalhar necessidades (tamanho, comodos, acabamentos, estilo)
2. Desenvolver planta arquitetonica personalizada no AutoCAD (~1 semana)
3. Apresentar proposta ao cliente com renderizacoes 3D (Enscape/Lumion)
4. Coletar feedback e revisar planta (processo pode ter multiplas rodadas)
5. Definir allowances e "by owner" com Camila antes de fechar budget
6. Enviar projeto para engenheiro externo (retorna projeto executivo em PDF)
7. Entregar Design Book + floor plan + spec sheet ao Budget

**Handoff para proxima etapa:**
- Design Book, floor plan, spec sheet entregues ao Orcamento (Samuel/Felipe)
- Projeto executivo entregue ao Takeoff (Gabriela) para levantamento de quantidades
- Informacoes do projeto entregues a Client Relations para comunicacao com cliente

## Etapa 3: Orcamento - Estimativa e Budget Customizado

| Campo | Detalhe |
|-------|---------|
| **Departamento** | Orcamento/Budget (Felipe Nery / Samuel Torres) |
| **Gatilho de inicio** | Recebimento do Design Book + Takeoff do Escritorio de Projetos |
| **Ferramentas** | Planilhas Excel, Builder Trend, Google Drive |

**Acoes principais:**
1. Receber Design Book + Takeoff como base para o orcamento
2. Solicitar cotacoes individuais para cada novo projeto (minimo 3 fornecedores)
3. Detalhar itens e custos personalizados (piscina, acabamentos especiais, upgrades)
4. Definir allowances e "by owner" com Camila
5. Samuel e Camila aprovam budget inicial (R3 — estimativa condensada para cliente)
6. Apresentar budget ao cliente com Camila (categorias, nao item por item)
7. Apos aprovacao do cliente: solicitar contrato de construcao
8. Apos contrato assinado: PMs (Daniel, Cris, Edson) aprovam budget real com cotacoes de subs

**Nota:** Budget customizado nunca e definitivo — clientes mudam especificacoes durante a obra, exigindo revisao constante.

**Handoff para proxima etapa:**
- Budget aprovado pelo cliente entregue ao Financeiro e ao PM responsavel
- Cotacoes selecionadas entregues a Contratos (Bruna) como base para contratos com subs
- Contrato de construcao solicitado e preparado pelo Escritorio de Projetos / Permits

| **Ferramentas** | E-mail (canal formal), WhatsApp (subs), Builder Trend, planilha de tracking |

**Acoes principais:**
1. Registrar data de ocupacao como inicio da garantia
2. Receber solicitacoes de reparo do cliente SOMENTE por e-mail
3. Encaminhar para sub responsavel via WhatsApp (grupos por tipo: Eco One para encanamento, etc.)
4. Acompanhar execucao — cobrar a cada 3 dias sem resposta
5. 3 daily logs por reparo: open claim, schedule, finalize
6. Supervisor Daniel confirma reparos em visitas de rota
7. Controle de cortes de grama conforme necessario

**Fechamento do ciclo:**
- Periodo de garantia ativo por 1 ano a partir da data de ocupacao
- Sub responsavel tem garantia propria de 1 ano a partir do fim do servico
- Utilities transferidas ao proprietario (agua, energia, gas)

### Duracao estimada por fase

| Fase | Duracao Tipica | Observacoes Custom |
|------|---------------|-------------------|
| Captacao + Pre-agreement | 2-8 semanas | Dependente do cliente, negociacao |
| Desenvolvimento do Projeto | 4-12 semanas | Revisoes com cliente podem estender |
| Orcamento + Aprovacao | 2-6 semanas | Cotacoes individuais demoram mais |
| Permits (custom) | 6-16 semanas | Mais complexo — documentos adicionais |
| Fase 1 (Fundacao) | 4-8 semanas | Similar ao spec |
| Fase 2 (Rough) | 6-10 semanas | Especificacoes customizadas complicam coordenacao |
| Fase 3 (Acabamento) | 8-16 semanas | Itens customizados (piscina, pavers) adicionam tempo |
| Inspecoes Finais + CEO | 1-3 semanas | Impact fees podem bloquear em Marion |
| Closing | 2-4 semanas | Mais detalhado — expectativas maiores do cliente |
| Warranty | 1 ano pos-ocupacao | PM Daniel acompanha mais de perto |

### Diferencas criticas: Custom vs Spec Home

| Aspecto | Spec Home | Custom Home |
|---------|-----------|-------------|
| Inicio do fluxo | Land Acquisition | Client Relations |
| Planta arquitetonica | Template existente | Criada do zero para o cliente |
| Orcamento | Template replicavel | Cotacoes individuais por projeto |
| Gestao da obra | Schedulers | PM dedicado (Daniel/Cris/Edson) |
| Compras | Setor de Compras (Brenda) | PM da custom gerencia diretamente |
| Mudancas durante obra | Raras (modelo padronizado) | Frequentes (cliente muda especificacoes) |
| Prazo medio | Menor | Maior (dependencia do cliente) |
| Complexidade de permits | Padronizada | Maior (documentos adicionais, HOA, etc.) |
| Budget | Fechado e replicavel | Aberto — revisado constantemente |
| Warranty | Padrao | PM Daniel acompanha mais proximamente |
